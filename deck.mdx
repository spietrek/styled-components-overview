# Component Based Design

1. Split UI into small, more manageable parts.
2. Based on Atomic Design.
3. Isolating components and states allow developers to articulate behavior and functionality.
4. React is structured around components.

---

# Component Reusability

1. Always look for ways to reuse code between projects.
2. UI components are often designed with reusability in mind.
3. Code (including styles) is packaged together to make sharing easier and avoid unused code bloat.
4. Bit allows sharing components code between projects. All components are organized on the cloud, made discoverable, and synced to all your projects.

# Inline Styling

```
import React from 'react';

const divStyle = {
  margin: '40px',
  border: '5px solid pink'
};

const pStyle = {
  fontSize: '15px',
  textAlign: 'center'
};

const Box = () => (
  <div style={divStyle}>
    <p style={pStyle}>Get started with inline style</p>
  </div>
);

export default Box;
```

> "CSS-in-JS" refers to a pattern where CSS is composed using JavaScript instead of defined in external files.

# CSS-in-JS Benefits

1. Scoped styles without selectors. Unique class names are created to avoid collisions.
2. Avoids specificity conflicts. Two style definitions for one element can be used.
3. Source order independence. No need to worry about how the order of files is imported.
4. Dead code elimination. Linters will show unused components so they can be removed if not used.
5. Vendor prefixing. Adds vendor prefixes only for the required browser.
6. Highly expressive which makes reading JSX easier.

# styled-components Benefits

1. Use as as much or as little as you want
2. Use your favorite UI framework and add styled-components on top
3. Defining styled components that possess their own encapsulated styles
4. Layout and styling in a single file for efficiency
