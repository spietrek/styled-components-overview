# Styling in ReactJS

## CSS Stylesheet
## Inline Styling
## CSS Modules
## CSS-in-JS

<Notes>
1. BEM, LESS, Sass did not provide true encapsulation. CSS Modules does.
2. It handles creating dynamic class names to avoid conflicts.
3. CSS Modules only show you a way of localizing style definitions: a clever way of automating BEM so you don’t need to think about chosing a class name ever again (or at least think about it less).
</Notes>

---

> Which ever technology (SASS, Less, BEM, CSS Modules, CSS-in-JS) you choose, there is no substitute for a well-defined styling architecture.

---

> CSS-in-JS refers to a pattern where CSS is composed using JavaScript instead of defined in external files.

<br />

## - ReactJS Docs

---

| CSS Challenges | CSS-in-JS Benefits |
| ----------------| ------------- |
| Global Namespace | Unique class names are automatically created to avoid collisions |
| Dependencies | CSS is encapsulated |
| Ordering of CSS | No issues with order |
| Dead Code | Linters can show unused code |
| Reusable Code | CSS is defined in *components* which can be reused |
| Difficult to Scale | Break code (including CSS) down into components |

# Component Based Design

1. Aka Component Driven Development
2. Promote reusability
3. Split UI into small, more manageable parts
4. Focused development
5. Developed independently using contract specfications
6. Code (including styles) is packaged together
7. Targeted feedback
8. Refactored or replaced

<Notes>
Allow developers to articulate behavior and functionality.
</Notes>

---

# Design Systems

1. Pattern Libraries can be used to style components using Atomic Design.
2. Style guides, like Storybook or Styleguidist, allow to work on a component, including business logic and tests, in isolation.
3. [Bit](https://bitsrc.io/) allows sharing component code between projects where components are stored on the cloud.

---

<Image src="images/styled-components.png" size="30%"/>

<Notes>
styled-components Benefits

1. Use as as much or as little as you want.
2. Props instead of classes.
3. Defining styled components that possess their own encapsulated styles.
4. Layout and styling in a single component for efficiency.
5. Highly expressive which makes reading JSX easier (JS + HTML + CSS).

</Notes>

# styled-components V4

* Smaller and much, much faster
* A refreshed createGlobalStyle API, replacing the old injectGlobal
* Support for the "as" prop, a dynamic alternative to .withComponent
* Removal of Comp.extend in favor of unifying styled(Comp)
* Full StrictMode compliance for React v16
* Native support for ref on any styled component

<Notes>
1. You can nest rules (pseudo selectors, media queries, parent components).
2. Integrate existing CSS frameworks using .attrs.
3. Use .attrs to ensure every instance of a style component has the same props.
4. If needed, you can override specificity using &&& (not important).
5. Override inline using &[style] {}.
6. Browsers: v2.x IE9+, v3.x IE9+, v4.x IE11 (IE9+ with Map + Set polyfills) -> all evergreen browsers
</Notes>
