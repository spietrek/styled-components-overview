# Inline Styling

```
import React from 'react';

const divStyle = {
  margin: '40px',
  border: '5px solid pink'
};

const pStyle = {
  fontSize: '15px',
  textAlign: 'center'
};

const Box = () => (
  <div style={divStyle}>
    <p style={pStyle}>Get started with inline style</p>
  </div>
);

export default Box;
```

---

> "CSS-in-JS" refers to a pattern where CSS is composed using JavaScript instead of defined in external files.

<br />

## - ReactJS Docs

---

| CSS Challenges | CSS-in-JS Benefits |
| ----------------| ------------- |
| CSS development too slow | Full CSS support. Themes. Editor syntax highlighting. Automatic vendor prefixing. |
| Global namespace, Styling conflicts | Optional global CSS. Scoped styles without selectors. Unique class names are created to avoid collisions. |
| Specificity | Two style definitions for one element can be used to avoid specificity conflicts. |
| Hardly reusable | CSS is defined in *components* which can be reused. |
| Dead code | Linters can show unused CSS. |

# Component Based Design

1. Developers should always look for ways to reuse code between projects.
2. Components promote reusability.
3. Components can be split UI into small, more manageable parts.
4. Isolating components and states allow developers to articulate behavior and functionality.
5. Components can be developed independently using contract specfications.
6. Component code (including styles) is packaged together to make sharing easier and avoid unused code bloat.
7. Components can be refactored or replaced.
8. Style guides, like Storybook or Styleguidist, allow to work on a component in isolation.
9. [Bit](https://bitsrc.io/). Allows sharing components code between projects. All components are organized on the cloud, made discoverable, and synced to all your projects.

# styled-components Benefits

1. Use as as much or as little as you want.
2. Props instead of classes.
3. Defining styled components that possess their own encapsulated styles.
4. Layout and styling in a single component for efficiency.
5. Highly expressive which makes reading JSX easier (JS + HTML + CSS).
